# Публикация вашего мода на Modtale

[Modtale](https://modtale.net) — современная платформа для обмена и поиска Hytale модов, плагинов и другого контента. Данное руководство покажет полноценный процесс публикации своего мода.

## Требования

Прежде чем начать, убедитесь, что у вас есть:

- Скомпилированный файл `.jar` вашего мода
- Высококачественные скриншоты или изображения вашего мода в действии
- Логотип или значок (рекомендуемый размер: 512x512px)
- Подробное описание функций вашего мода
- Задокументированные зависимости или требования

## Пошаговый процесс публикации

### 1. Настройки учетной записи

Войдите в [Modtale](https://modtale.net), используя предпочтительный метод. Если у вас еще нет аккаунта, создайте его!

### 2. Начало процесса

Нажмите на **"Создать"**  (Create) на панели навигации или перейдите на [страницу загрузки](https://modtale.net/upload).

### 3. Выберите тип контента

Выберите подходящий тип контента для вашей публикации:

- **Server Plugins** — для Hytale server-plugins и модов
- **Data Assets** — для пользовательских наборов данных, loot-tables и другого контента на основе данных
- **Art Assets** — для наборов текстур (текстурпак), моделей и другого художественного контента
- **Worlds** — пользовательские миры и схематики
- **Modpacks** — для коллекций модов и др. контента

![Выбор типа содержимого](/assets/guides/modtale_type.png)

### 4. Основная информация

Заполните необходимые данные:

- **Название**: Выберите понятное, описательное название для вашего мода
- **Описание**: Напишите краткое описание (1-2 предложения), которое объясняет, что делает модификация.
- **Теги**: Добавьте соответствующие теги, чтобы помочь пользователям найти ваш мод.

Название и описание проекта

### Project Configuration

Now you're on your project's management page. Here you can:

#### Upload Files

Navigate to the **"Files"** tab and upload your `.jar` file:

- Drag and drop your file or use the upload button
- Add version notes describing changes and new features
- Mark release type (Release, Beta, Alpha)

![File Upload Interface](/assets/guides/modtale_upload.png)

#### Customize Your Project

- **Logo**: Upload a high-quality logo (PNG format recommended)
- **Screenshots**: Add multiple screenshots showing your mod in action
- **Description**: Write a comprehensive description using Markdown formatting
- **Links**: Add links to your source code, Discord server, or documentation

## Best Practices

### Documentation

<Callout type="info">
  Информация Users should understand what your mod does, how to use it, and any special requirements.
</Callout>

### Version Notes

When uploading new versions:

- Clearly document all changes and new features
- Include bug fixes and known issues
- Mention any breaking changes or migration steps
- List new dependencies or requirement changes

### Visual Appeal

- Use high-quality screenshots that showcase your mod's features
- Create an eye-catching logo that represents your mod
- Consider adding a banner image for your project page

### Community Engagement

- Respond to comments and feedback promptly
- Consider creating a Discord server for community support
- Regularly update your mod based on user feedback

## После публикации

### Мониторинг вашего проекта

- Проверяйте аналитику, чтобы увидеть статистику скачиваний
- Отслеживайте комментарии и отзывы
- Следить за сообщениями об ошибках и запросами

### Marketing Your Mod

- Share your mod on relevant Hytale communities
- Create showcase videos or tutorials
- Engage with other modders and creators

<Callout type="warning">
  Remember to keep your mod updated for new Hytale versions and maintain compatibility with Hytale's modding framework to maximize your audience reach.
</Callout>

## Getting Help

If you encounter any issues during the publishing process:

- Join their Discord community for support
- Contact the Modtale team directly through their support channels

# Modellering / Texturkonst

Hytale bidrar med omfattande stöd för 3d modellering och skapandet av texturkonst med specialiserade verktyg och arbetsflöden designade att behålla spelets distinka visuella stil medan kreatörer stärks.

## Nuvarande Modverktyg

### Blockbench

Hytale använder **Blockbench** som det primära verktyget för 3D-modellering och animation:

* **Officiellt stöd**: Används för att "skapa Hytale-modeller, texturer och animationer"
* **Specieltl plugin**: Hytale kommer att ha ett dedikerat Blockbench plugin, som ska släppas så snart som möjligt
* **Direkt kompatibilitet**: Pluginen tillåter skapare att "skapa Hytale-kompatibla modeller, texturer och animationer direkt i Blockbench"
* **Ersätter äldre verktyg**: Ersätter helt den gamla Hytale Model Maker för att bättre stödja etablerade kreativa arbetsflöden i gemenskapen
* **Säkerhetsfunktioner**: Du kan bara lägga till saker som är kompatibla med motorn, så du behöver inte oroa dig för att förstöra något

## Modellbegränsningar

### Geometrityper

Den största majoriteten av modeller i Hytale är begränsade till specifika geometriska primitiv:

**3D-element**:

* Block
* Cuboider
* Lådor

**2D element**:

* Quads
* Rektanglar
* Plan (bestående av 2 trianglar)

### Designfilosofi

Dessa begränsningar tjänar flera syften:

* **Prestandaoptimering**: Säkerställer effektiv rendering och spelprestanda
* **Konsekvent konststil**: Bibehåller Hytales distinkta visuella estetik
* **Sömlös integration**: Gör så att mods känns mer sammanhållna med basspelet

## Pixeldensitetsstandard

Hytale använder olika pixeldensiteter för olika typer av innehåll:

### Standard densiteter

* **Block**: 32px densitet
* **Spelarmodeller**: 64px densitet
* **Mobs**: Sannolikt också 64px densitet
* **Spelarutrustning**: 64px densitet (vapen, kosmetik, rustning, etc.)

### Flexibilitet

* **Fel densitetshantering**: Användning av felaktig densitet kan orsaka att tillgångar visas i fel storlek, men kommer inte att orsaka importfel
* **Inga strikta begränsningar**: Pixelförhållanden, rotation och skalning är inte begränsade
* **Förlåtande system**: Verktygen är utformade för att vara flexibla samtidigt som kompatibiliteten upprätthålls

## Hytale Asset Editor

### In-Game Editor

Asset Editor är ett omfattande verktyg som kommer att levereras med spelet:

* **Levereras med spelet**: Tillgängligt för alla spelare vid lansering.
* **Externt verktygskrav**: Stödjer inte bildredigering, du måste använda externa verktyg för texturskapande
* **Redigering av datatillgångar**: Primär redigerare för att arbeta med datatillgångar
* **Expanderande funktioner**: Stöder för närvarande de flesta funktioner, men har begränsningar runt NPCs, världsgeneration, och interaktioner som kommer att expandera med tiden

### Partikelskapande

Asset Editor innehåller robusta verktyg för partikelskapande:

* **Skapande i spelet**: Partiklar skapas direkt i spelet med hjälp av Asset Editor
* **Industristandard**: Partikellogik är robust och liknar Unity/Unreal partikelsystem
* **Händelsedriven**: Partiklar kan skapas på händelser eller interaktioner (som att öppna dörrar)
* **Enhetsfäste**: Kan fästa partiklar till enheter med specifika fästpunkter
* **Dynamisk rörelse**: Stöder partiklar som rör sig med sina förankringspunkter
* **Tidsfördröjningar**: Stöder tidsbaserade partikeleffekter och förseningar

## Material

### Optimeringsfokus

Hytale har en annan metod till material jämfört med andra spelmotorer:

* **Föroptimerade**: Materialen är mycket optimerade och artister kan inte skapa anpassade material
* **Prestandaprioritering**: Denna begränsning säkerställer konsekvent prestanda över allt innehåll
* **Kurerat system**: Bibehåller visuell konsistens och maximerar verkningseffektiviteten

## Bästa praxis

### Tillgångsoptimering

Viktiga överväganden från Hytales utvecklingsteam:

* **Snål resursanvändning**: Var medveten om hur många block/fyrkanter du använder och texturfilsstorlekar
* **Undvik onödig belastning**: Målet är att förhindra onödigt resursinnehåll som kan påverka prestandan.
* **Effektivitetsfokus**: Balansera visuell kvalitet med tekniska begränsningar

### Framtida dokumentation

* **Bästa praxisguide**: En omfattande "bästa praxis" guide för konst och modellering kommer att släppas någon gång
* **Flexibla gränser**: De flesta kreativa mål kan uppnås utan att bryta något i systemet
* **Gemenskapsdrivna arbetsflöden**: Verktyg är utformade för att stödja etablerade kreativa arbetsflöden i modding gemenskapen

## Integration med utvecklingspipeline

Arbetsflödet för modellering och texturkonst integreras sömlöst med Hytales bredare utvecklingsekosystem:

* **Kompatibilitet för tillgångsredigerare**: Modeller skapade i Blockbench arbetar direkt med Hytale Asset Editor
* **Integrering av datatillgång**: 3D-modeller kan konfigureras och anpassas genom datatillgångar
* **Visuellt skriptningsstöd**: Modeller kan kontrolleras och animeras genom Visuell skriptning
* **Java plugin utökning**: Avancerade modellbeteenden kan implementeras via Java plugins
* **Partikelintegration**: Modeller kan ha partiklar fästa vid specifika punkter för förbättrade visuella effekter

Detta integrerade tillvägagångssätt säkerställer att 3D-modellering och texturkonst inte är isolerade kreativa processer, utan centrala komponenter i Hytales omfattande utvecklingsmiljö.

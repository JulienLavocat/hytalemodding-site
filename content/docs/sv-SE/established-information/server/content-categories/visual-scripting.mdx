# Visuell Skriptning

Hytale planerar att stötta visuell skriptning som en metod för att göra avancerad logik och kontrollflöde tillgängligt för en bredare publik.  Detta system kommer inte att finnas tillgängligt vid lanseringen, men bör vara tillgängligt strax efter lanseringen.

## Översikt

Visuell skriptning i Hytale är designat för att fungera som Blueprints i Unreal Engine. Det tillåter designers att skapa komplexa beteende genom ett visuellt gränssnitt som matchar hur de tänker kring spellogik, utan behovet av att skriva kod direkt.

### Nyckelfuntkioner

* **Delar av Javafunktionalitet**: Visuell skriptning kan göra en delmängd av vad som är möjligt med Java plugins
* **Designervänligt**: "Designers får kraft genom ett visuellt språk som matchar hur de tänker om beteenden"
* **Utbyggbart för programmerare**: Programmerare kan utöka och optimera systemet genom att lägga till nya visuella noder och prestandakänsliga implementationer
* **Säker delningsmiljö**: Bidrar med en sanlådad, säker miljö för att dela logik mellan moddare

## Fördelar mot traditionell kodning

Visuell skriptning gör skrivandet av anpassad logik mer tillgänglig genom att ta bort vanliga hinder man kan hitta när man utvecklar i Java.

* **Ingen komplex konfiguration av miljö**: Ingen anledning att oroa sig om att installera rätt verktyg eller att konfigurera utvecklingsmiljöer
* **Syntaxfri utveckling**: Tar bort behovet av att lära sig och komma ihåg programmeringssyntax
* **Säkrare delning**: Skadlig programvara i visuellt skript bör vara nästan omöjlig
* **Mer inklusiv**: Sänker skicklighetsgränsen för icke-programmerare

## Relation med Java

Visuell skriptning och Java plugin är designade att arbeta tillsammans med varandra:

* **Java utökar Visuell skriptning**: Programmerare kan skapa nya visuella noder med hjälp av Java
* **Prestandaoptimering**: Medans Visuell skriptning kanske inte är lika optimerat som Java, programmerare kan implementera prestandakritisk logik i Java och exponera det som visuella noder
* **Bästa av båda världar**: Tillåter majoriteten av logik att bevaras synligt i den tillgänliga Visuella skriptningsmiljön medan den behåller dyra operationer optimerade.

Hytale rekommenderar starkt Visuell skriptning eftersom det är både mer tillgänligt och säkrare att dela och ladda ner. Men att stanna i Java kommer fortfarande funka för de som föredrar det.

## 3D Visuell Skriptning

En av Hytales unika funktioner är att komma med Visuell skriptning direkt in i 3D världen:

* **I världs redigering**: Länkspakar, dörrar, spawners, triggers och andra element direkt i spelet
* **Skapandet av äventyrskartor**: Utmärkt för att bygga äventyrskartor och bemötanden utan att lämna Hytale
* **Liknande DOOM SnapMap**: Bidrar med ett välbekant arbetsflöde för dem som har använt liknande system
* **Fullt integrerat**: Fungerar sömlöst tillsammans med både asset-flödet och skriptstacken

## Ingen textbaserad Skriptning

Hytale har gjort ett medvetet beslut om att inte inkludera textbaserade skriptspråk. Deras resonemang:

* **Falska kompromisser** Textbaserad skriptning är ofta introducerat för att "hjälpa ickeprogrammerare" men kräver fortfarande programmeringskoncept
* **Svårighetsökning**: Programmerare slutar med att jonglera mellan två språk där ett är mindre kraftfullt
* **Kräver fortfarande syntax**: Designers behöver fortfarande lära sig syntax och programmeringskoncept
* **Mindre inklusivt**: Löser inte helt tillgänglighetsproblem
* **Fragmentering**: Undviker "att fragmentera logik mellan flertalet 'halvspråk'"

Istället, Hytale tror att Visuell skriptning är en bätre metod som faktiskt gör spellogik tillgängligt för en bredare publik samtidigt som den bibehåller den kraft och flexibilitet som behövs

# Bidra till HytaleModding

Vi välkomnar bidrag från gemenskapen att hjälpa till att förbättra och exapndera HytaleModding dokumentationen. Oavsett om du fixar felstavningar, lägger till nya guider, eller bidrar med kodexempel så är dina bidrag värdefulla!

## Hur du kan bidra

### Rapportera Problem

Om du har funnit ett fel med dokumentationen, var vänlig och anmäl det genom att öppna ett nytt problem på vår [GitHub repository](https://github.com/HytaleModding/site/issues).

Om du kan arbeta på detta problemet själv, kan du be om att bli tillsatt det och skicka in ett PR.

### Skicka in Pull Requests

1. Forka förrådet på GitHub.
2. Klona din fork till din lokala maskin.
3. Skapa en ny gren för dina ändringar.

```bash
git checkout -b my-feature-branch
```

4. Gör dina ändringar och begå dem med tydliga meddelanden. Vi följer [konventionella åtaganden](https://www.conventionalcommits.org/en/v1.0.0/), din PR kommer inte att accepteras om dina incheckningsmeddelanden inte följer denna konvention.

```bash
git commit -m "Add feature X to documentation"
```

5. Pusha dina ändringar till din fork.

```bash
git push origin my-feature-branch
```

6. Öppna en Pull Request på det ursprungliga förrådet, beskriver dina ändringar och varför de bör slås samman.

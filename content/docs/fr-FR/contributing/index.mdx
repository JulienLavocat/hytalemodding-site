# Contribuer à HytaleModding

Nous accueillons les contributions de la communauté pour aider à améliorer et étendre la documentation d'HytaleModding.  Que vous corrigiez des typos, ajoutiez des nouveaux tutoriels, ou fournissiez des exemples de code, votre contribution est précieuse !

## Comment contribuer

### Signaler des problèmes

Si vous avez trouvé un problème avec la documentation, signalez-le en ouvrant une issue sur notre [dépôt GitHub](https://github.com/HytaleModding/site/issues).

Si vous pouvez travailler sur ce problème vous-même, vous pouvez demander à y être assigné et soumettre une PR.

### Soumettre une Pull Request

1. Forkez le dépôt sur GitHub.
2. Clonez votre fork sur votre machine locale.
3. Créez une nouvelle branche pour vos changements.

```bash
git checkout -b my-feature-branch
```

4. Faites vos changements et commitez-les avec un message clair. Nous utilisons la convention [conventional commits](https://www.conventionalcommits.org/en/v1.0.0/), votre PR ne sera pas acceptée si vos messages de commits ne suivent pas cette convention.

```bash
git commit -m "Add feature X to documentation"
```

5. Pushez vos changements dans votre fork.

```bash
git push origin my-feature-branch
```

6. Ouvrez une Pull Request sur le dépôt original, en décrivant vos changements et pourquoi ils devraient être fusionnés.
